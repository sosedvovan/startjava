установи git.exe (локальный репозиторий) на компе
зарегистрируйся на github и создай там (облачный)репозиторий (например с названием startjava)
создай на диске D: папку с именем как и у репозитория  (startjava)
в терминале перейди к этой папке (startjava)- ПКМ + shift --> открыть окно а PowerShell здесь
и нам  надо будет зарегистрировать эту папку локальном репозитории  git.exe командой     --- git init  -- 
и зарегистрировать (привязать) локальный репозиторий к облачному github командой  
---  git remote add origin https://github.com/sosedvovan/startjava.git---
причем url(это путь к репозиторию на облаке который): --origin https://github.com/sosedvovan/startjava.git --
 мы взяли со страници создания репозитория: --https://github.com/sosedvovan/startjava --
причем origin- это псевдоним url-а (будем его использовать для команды отправки в облако (запушить))  
чтобы когда пушить длинный url не писать(можно произвольно но обычно пишут origin)
----git remote -v ---- команда отобразит что находится на облачном репозитории для проверки сделанного

команда ---ls ---  это чтобы посмотреть что лежит в репозитории startjava (кроме скрытых?)


ИНДЕКСИРУЕМ
допустим там лежит папка Lesson_1 и в ней файлы с расширением java
чтобы отправить содержимое папки Lesson_1 в индекс git-локальный чтобы он их стал отслеживать:
команда: ---!!!!!!!! git add "lesson_1/"  !!!!!!!!!!!---
чтобы git проигнороровал и не загружал файлы с расширением .class (кот тоже в папке lesson_1) в индекс, надо ввести команду:
git reset "lesson_1/имя файла который надо проигнороровать" можно так: ---git reset "lesson_1/*.class" ---- 
теперь git эти файлы отслеживает(красным) но не добавляет в индекс
и после проверить статус репозитория командой: ----!!!!!!!!!!  git status   !!!!!!!!!!!!------
github подтвердит что видит папку lesson_1 и ее содержимое(файлы .java)


КОММИТЕМ
далее надо сделать слепок состояния кода в startjava и отправить(заккомитить) этот слепок в наш локальный репозиторий git.exe командой:
---!!!!!!!!!!!!!!!!! git commit -m "implement HelloWorld"  !!!!!!!!!!!!----  где -m "implement HelloWorld" это комментарий к слепку
и после проверить статус репозитория командой: ----  git status   ------
git ответит что больше нечего коммитить, те все файлы уже в слепке и отправленны

у меня git попросил еще ввести мою почту и имя:
PS D:\startjava>  git config --global user.email "drugst@yandex.ru"
PS D:\startjava> git config --global user.name "Vladimir"

то мы на локальном репозитории зафиксировали (заккомитили) код который на данный момент имеем.

отобразим информацию о нашем локальном коммите командой:
---!!!!!!!!!! git log   !!!!!!!!!!!---


ПУШИМ
теперь коммит с локального репозитория надо отправить (запушить) на облачный репозиторий командой:
!!!!!!!!!!!!! git push -u origin master  !!!!!!!!!!!!!!!!!!
где origin это псевдоним url-а облачного репозитория
а master - это поток...

у меня запустилась аунтификация через браузер-- я ввел пароль от github.com




ПОБЫРИКУ
git status  --если красные файлы- надо их заиндексировать на локальный:
git add "lesson_1/" 
git status -еще паз проверим-файлы проиндексировались и стали зелеными, теперь их закомитим:
git commit -m "implement HelloWorld"
git log -проверим состояние локального git и запушим на облачный github:
git push -u origin master 



----git remote -v ---- команда отобразит что находится на облачном репозитории

-----  git push -f ---- перезаписать все что есть в облаке тем, что только что закомитил



Я случайно занес в индекс git папку out с class-файлами. И, если я не уберу ее,
то при пуше она уйдет на github. Как сделать так что бы git ее не видел и не
индексировал?
Написать в консоли --------- git rm -r --cached out -------, а после занести папку out в
файл .gitignore
